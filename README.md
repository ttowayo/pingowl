# 🦉 PingOwl (Site Monitoring Dashboard)

PingOwl은 웹사이트와 API의 상태를 실시간으로 감시하고 시각화하는 모던한 사이트 모니터링 대시보드입니다.

## 🚀 주요 기능

### 1. 실시간 모니터링 및 상태 표시

- 웹사이트의 **온라인/오프라인 상태**를 실시간으로 확인합니다.
- **응답 속도(ms)**를 측정하여 사이트의 성능을 모니터링합니다.
- 상태에 따라 직관적인 컬러(정상, 지연, 에러)로 표시됩니다.

### 2. 강력한 체크 모드

- **WEB 페이지 체크**: 특정 페이지에 접속하여 원하는 **키워드**가 존재하는지 확인합니다. (예: "로그인" 버튼이 제대로 로드되는지 확인)
- **API 접속 체크**: API 엔드포인트가 정상적으로 **HTTP 200** 응답을 반환하는지 체크합니다.

### 3. 응답 트렌드 시각화 (Sparkline)

- 최근 15회의 모니터링 기록을 **스파크라인 그래프**로 보여줍니다.
- 응답 시간의 변화를 한눈에 파악하여 간헐적인 장애나 성능 저하를 감지할 수 있습니다.

### 4. 백그라운드 자동 감시

- 서버가 실행 중인 동안 **5분마다 자동으로** 모든 등록된 사이트를 체크합니다.
- 모니터링 결과는 데이터베이스에 저장되어 히스토리로 관리됩니다.

### 5. 사용자 관리 및 데이터 동기화

- **Supabase Auth**를 연동하여 안전한 회원가입 및 로그인을 제공합니다.
- 사용자별로 모니터링 대상 사이트를 별도로 관리하며, 데이터는 **Supabase DB**에 실시간 저장됩니다.

## 🛠 작동 방식

### 시스템 아키텍처

1. **Frontend (Vite + Vanilla JS)**:
   - 사용자 인터페이스를 담당하며, 백엔드 API와 통신하여 데이터를 렌더링합니다.
   - 드래그 앤 드롭을 통한 순서 변경, 실시간 차트 렌더링 등의 기능을 포함합니다.

2. **Backend (Express + Node.js)**:
   - 외부 사이트 체크 시 발생하는 CORS 문제를 해결하기 위한 **프록시 서버** 역할을 수행합니다.
   - 주기적으로 실행되는 **백그라운드 모니터링 로직**이 포함되어 있습니다.

3. **Database & Auth (Supabase)**:
   - 사용자의 사이트 설정 정보 및 모니터링 히스토리 데이터를 저장합니다.
   - JWT 기반의 인증 시스템을 통해 데이터 보안을 유지합니다.

### 실행 방법

1. **의존성 설치**

   ```bash
   npm install
   ```

2. **환경 변수 설정**
   `server/.env` 파일에 Supabase 설정 정보를 입력합니다.

   ```env
   SUPABASE_URL=your_supabase_url
   SUPABASE_ANON_KEY=your_supabase_anon_key
   ```

3. **개발 서버 실행**

   ```bash
   npm run dev
   ```

   - 백엔드 서버: `http://127.0.0.1:3001`
   - 프론트엔드 서버: `http://127.0.0.1:5600` (Windows 예약 포트 회피를 위해 5600 사용)
